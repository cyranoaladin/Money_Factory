Voting Power Formula (On-Chain)

pub fn calculate_voting_power(
    staked_mfai: u64,
    proof_of_skill_nfts: Vec<Pubkey>,
    participation_score: u64,
) -> u64 {
    let vp_staked = staked_mfai; // 1:1 ratio
    let vp_skills = (proof_of_skill_nfts.len() as u64) * 50;
    let vp_reputation = participation_score / 2;
    
    vp_staked + vp_skills + vp_reputation
}

Collaterize Bonding Curve Mechanics

Formula (Bancor-inspired):

Token Price = Reserve Balance / (Token Supply × Reserve Ratio)

Buy Price = Reserve / ((Supply + Amount) × Ratio)
Sell Price = Reserve / ((Supply - Amount) × Ratio)

Parameter Ranges

Parameter Range Impact
Reserve Ratio 0.2 - 0.5 Lower = steeper curve, higher volatility
Higher = gentler curve, more stable
Initial Reserve 50k - 500k USDC Based on project size and DAO approval
Buy Fee 1-3% 60% Treasury, 20% Creator, 20% LP rewards
Sell Fee 1-3% Same allocation as buy fee

Example Launch

  • Project: DeFi lending protocol
  • Reserve: 200,000 USDC
  • Ratio: 0.35
  • Initial Supply: 1,000,000 tokens
  • Starting Price: 200,000 / (1,000,000 × 0.35) = $0.571 per token

Rug-Pull Prevention

  • Reserve locked in smart contract (no admin withdrawal keys)
  • Mathematical price discovery (no manipulation possible)
  • Transparent on-chain reserves (verifiable by anyone)
  • Liquidity lock periods enforced (6-12 months minimum)
Graduation Criteria:
  • Mainnet deployment successful (verified contracts)
  • Token tradeable on ≥2 DEXs (Raydium + Orca)
  • Bonding curve active with initial liquidity ($50k+ reserve)
  • Community governance transitioned (token distribution >50% public)

5.2 Collaterize Bonding Curve Deep Dive

Mathematical Foundation

Price(t) = Reserve(t) / (Supply(t) × Reserve_Ratio)

When user BUYS x tokens:
  Reserve_new = Reserve_old + (x × Price_avg)
  Supply_new = Supply_old + x
  Price_new = Reserve_new / (Supply_new × Ratio)

When user SELLS x tokens:
  Supply_new = Supply_old - x
  Reserve_new = Reserve_old - (x × Price_avg)
  Price_new = Reserve_new / (Supply_new × Ratio)

Parameter Impact Analysis

Reserve Ratio Characteristics Use Case
0.2 (Steep) High volatility, faster price growth
10% supply increase → 12.5% price increase		 High-growth speculative projects
0.5 (Gentle) Low volatility, stable pricing
10% supply increase → 5% price increase		 Mature, utility-focused projects
